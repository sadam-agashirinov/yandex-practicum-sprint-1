Для перехода на микрофронтенд выбрал Webpack Module Federation.
Тип интеграции - Build time.
Тип композиции - клиентский.

Почему так:
1. Приложение содержит одну страницу на которой представлен весь функционал.
	Можно собрать сразу всю страницу и показать его клиенту. Фунционал не большой поэтому вес конечной сборки будет не большой.
2. Весь фронт использует один фреймворк - React. Использовать несколько JavaScript-фреймворков на одной странице - преимущество SPA. Нам это здесь не нужно.
3. Возможность использовать общий код во время выполнения.



Микрофронты.

0. host - Главный компонент объединяющий остальные Микрофронты.
1. mf-auth - микрофронтенд отвечающий за авторизацию и регистрацию.
2. mf-profile - микрофронтенд отвечающий за работу с профилем пользователя.
3. mf-card - микрофронтенд отвечающий за работу с карточками
4. shared - папка содержащая общий компоненты используемые Микрофронтами.
